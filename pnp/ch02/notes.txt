Review Questions

1. What is meant by the term prompt?
   A message shown to the user asking for input

2. Which operator do you use to read into a variable?

> > (used with std::cin)

3. What notations can you use to initialize an object?
   =, (), {}, = {}

4. Get a number from the user and put it into a variable.

int number;
std::cout << "Enter a number: ";
std::cin >> number;

5. What is \n called and what purpose does it serve?
   Newline character; moves the cursor to the next line

6. What terminates input into a string?
   Whitespace (space, newline, tab)

7. What terminates input into an integer?
   Whitespace or invalid (non-numeric) input

8. How would you write the following as a single line?
   std::cout << "Hello " << first_name << '\n';

9. What is an object?
   A region of memory with a type and a value

10. What is a literal?
    A value written directly in the code

11. What kinds of literals are there?
    int, double, char, string, bool

12. What is a variable?
    A named object

13. What are typical sizes for a char, an int, and a double?
    char = 1 byte, int = 4 bytes, double = 8 bytes

14. What measures do we use for the size of small entities in memory, such as ints and strings?
    Bytes (and bits)

15. What is the difference between = and ==?
    = is assignment, == is equality comparison

16. What is a definition?
    Creating a variable

17. What is an initialization and how does it differ from an assignment?
    Initialization gives a value when the variable is created; assignment gives a value after creation

18. What is string concatenation and how do you make it work in C++?
    Joining strings together using the + operator

19. What operators can you apply to an int?
    +, -, *, /, %, ++, --, >, <, >=, <=

23. What is type safety and why is it important?
    Ensures values match their types; prevents errors and undefined behavior

24. Why can conversion from double to int be a bad thing?
    Information loss (narrowing conversion)

25. Define a rule to help decide if a conversion from one type to another is safe or unsafe.
    Safe if no information is lost and no overflow occurs; unsafe if narrowing or overflow happens

26. How can we avoid undesirable conversion?
    Use {} (list initialization)

27. What are the uses of auto?
    Type deduction and simpler code for complex types
