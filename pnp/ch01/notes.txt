C++ Chapter 1 – Clean Notes (P&P)

Core Principles

* Programming is learned by writing programs
* A program is a sequence of instructions for the computer
* Execution always starts from the main() function

Program Lifecycle (very important)

1. Source code (.cpp file)
2. Compilation → object file (.o / .obj)
3. Linking → executable
4. Execution → OS loads program into memory → starts at main()

Compilation

* Done by the compiler (g++)
* Converts .cpp → .o
* Checks syntax (grammar of C++)

Linking

* Done by the linker
* Combines object files and libraries
* Resolves missing functions and references

Execution

* OS loads the executable into memory
* Program starts running at main()
* CPU executes instructions

File Types

* .cpp → source file (compiled)
* .h / .hpp → header file (included, not compiled directly)
* .o / .obj → object file (compiled code)
* Executable → final runnable program

Portability

* Source code → portable (if written properly)
* Object files → NOT portable
* Executables → NOT portable (OS + architecture dependent)

Function Structure

1. Return type
2. Function name
3. Parameters
4. Function body

main() Function

* Required in every C++ program
* Entry point of execution

return 0;

* Indicates successful execution

#include

* Includes declarations from other files/libraries

Comments

* // single-line comment
* /* multi-line comment */
* Ignored by compiler, used for notes

Compilation Command Example
g++ -std=c++20 -Wall main.cpp -o program

* g++ = compiler
* -std=c++20 = language version
* -Wall = show warnings
* main.cpp = source file
* -o program = output executable
